El proyecto sigue una arquitectura **modular inspirada en Domain-Driven Design (DDD)**, organizada en capas bien definidas dentro del directorio `app/`. Cada módulo de negocio encapsula sus propias entidades, casos de uso, repositorios e infraestructura, mientras que el código transversal reside en `app/Shared/`.

## Capas de la arquitectura

Cada módulo en `app/Modules/` replica la misma estructura de tres capas:

<Accordion title="Domain">
  Contiene la lógica de negocio pura, sin dependencias externas:

  - **Entity/**: Entidades de dominio con identidad propia.
  - **ValueObject/**: Objetos de valor inmutables.
  - **Repository/**: Interfaces de repositorios (contratos, no implementaciones).
  - **Enum/**: Enumeraciones de dominio.
  - **Exception/**: Excepciones de dominio específicas del módulo.
  - **Service/**: Servicios de dominio sin efectos secundarios de infraestructura.

  Ejemplo: `app/Modules/User/Domain/`
</Accordion>

<Accordion title="Application">
  Orquesta los casos de uso coordinando el dominio con la infraestructura mediante interfaces:

  - **UseCase/**: Clases de un solo método público `execute()`. Cada clase representa una acción de usuario.
  - **DTO/**: Objetos de transferencia de datos para entrada y salida de casos de uso.

  Ejemplo: `app/Modules/User/Application/UseCase/CreateUserUseCase.php` — utilizado por el comando CLI `importar-usuarios.php`.
</Accordion>

<Accordion title="Infrastructure">
  Implementaciones concretas de las interfaces de dominio: repositorios PDO, servicios de correo, etc.

  La infraestructura compartida reside en `app/Infrastructure/` y cubre:
  - PDO (MySQL con `utf8mb4`)
  - PHPMailer con cola de correo en base de datos
  - Motor de plantillas Latte 3
  - Gestión de sesión PHP nativa
  - Tokens CSRF con expiración de 1 hora
</Accordion>

## Bootstrap y contenedor DI

La clase `Bootstrap` en `app/Bootstrap.php` es el único punto de arranque del sistema. Se invoca en cada `*.php` del directorio `public/`:

```php
$container = Bootstrap::buildContainer();
```

El proceso de arranque realiza:

1. **Carga variables de entorno** con `vlucas/phpdotenv`, validando las claves requeridas (`DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `MAILER_HOST`, `MAILER_USER`, `MAILER_PASSWORD`).
2. **Detecta el entorno** a través de `APP_ENV` (`dev` o `prod`).
3. **Construye el contenedor** PHP-DI. En producción activa la compilación en `tmp/di_cache` y la escritura de proxies en `tmp/di_proxies`.
4. **Registra los módulos** — cada módulo implementa `ModuleInterface` y registra sus propias definiciones:

```php
$modules = [
    new AuthModule(),
    new MessagingModule(),
    new PublicationModule(),
    new SettingModule(),
    new UserModule(),
];

foreach ($modules as $module) {
    $module->register($builder);
}
```

5. **Carga los archivos de configuración** `config/settings.php` y `config/definitions.php`.
6. **Aplica la configuración de sesión** (strict mode, secure cookie, httponly, SameSite Lax).

## Inyección de dependencias (PHP-DI)

El contenedor se configura en `config/definitions.php`. Las dependencias clave y sus vinculaciones son:

```php
// Motor de plantillas Latte
Engine::class => function (AppConfig $config, LatteExtension $ext) {
    $latte = new Engine();
    $latte->setCacheDirectory(__DIR__ . '/../tmp/latte_cache');
    $latte->setAutoRefresh($config->isDev);
    return $latte;
},

// Conexión PDO (MySQL utf8mb4)
PDO::class => function (AppConfig $config) {
    $dsn = "mysql:host={$config->database->host};dbname={$config->database->database};charset=utf8mb4";
    return new PDO($dsn, $config->database->user, $config->database->password, [
        PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]);
},

// Interfaces y sus implementaciones concretas
RendererInterface::class         => autowire(LatteRenderer::class),
SessionInterface::class          => autowire(PhpSession::class),
MailerInterface::class           => autowire(DatabaseMailQueue::class),
TransactionManager::class        => autowire(PdoTransactionManager::class),
UserContextInterface::class      => autowire(UserContext::class),
UserProviderInterface::class     => get(UserContextInterface::class),
```

`AppConfig` se construye en `config/settings.php` leyendo variables de entorno a través de `EnvironmentInterface`, agrupando la configuración de base de datos, mailer, uploads y sesión en objetos de valor tipados.
