El sistema SIUT-ITSM está organizado en dos grandes áreas bien diferenciadas: una **zona pública** accesible sin autenticación y un **portal privado** que requiere sesión activa.

## Zona pública (`/`)

La zona pública es servida desde `public/index.php` mediante el motor de plantillas Latte. No requiere autenticación y expone las siguientes páginas:

| Página | Archivo | Descripción |
|--------|---------|-------------|
| Inicio | `public/index.php` | Página principal del sindicato |
| Acerca | `public/acerca.php` | Información institucional del SIUT-ITSM |
| Publicaciones | `public/publicaciones/` | Noticias y avisos públicos |
| Transparencia | `public/transparencia.php` | Información de transparencia sindical |
| Contacto | `public/contacto.php` | Formulario de contacto |
| Dudas | `public/dudas.php` | Preguntas frecuentes |

<Note>
  Las páginas de la zona pública usan la plantilla base `templates/@landing.latte`, que incluye el encabezado `@landing.header.latte` y es diferente del layout del portal privado.
</Note>

## Portal privado (`/portal/`)

El portal privado requiere autenticación. Cada petición a una página del portal pasa por el middleware `AuthMiddleware`, que redirige a `/cuentas/login.php` si la sesión no está activa. Si el usuario no tiene los permisos de rol requeridos, el `RoleMiddleware` redirige a `/portal/acceso-denegado.php` con HTTP 403.

La ruta de entrada es `public/portal/index.php`, que obtiene el usuario autenticado desde `UserProviderInterface` y renderiza `index.latte` con los datos de sesión.

## Flujo de autenticación

Cada archivo `*.php` en el portal privado sigue el mismo patrón de arranque:

```php
require_once __DIR__ . "/../../bootstrap.php";

$container = Bootstrap::buildContainer();

$middleware = $container->get(MiddlewareFactory::class);
$runner     = $container->get(MiddlewareRunner::class);

$runner->runOrRedirect($middleware->auth());
```

Este fragmento construye el contenedor DI, obtiene el factory de middlewares y ejecuta el guard de autenticación antes de que se procese cualquier lógica de negocio. Para rutas que requieren un rol específico se usa `$middleware->role(RoleEnum::Finanzas)` o cualquier combinación de roles.
