---
title: Roles y permisos
description: Descripción de los cinco roles de usuario del sistema y los recursos a los que cada uno tiene acceso dentro del portal.
---

El sistema gestiona el control de acceso mediante roles asignados a cada usuario en la columna `role` de la tabla `users`. El enum `RoleEnum` en `app/Shared/Domain/Enum/RoleEnum.php` define los valores válidos.

```php
enum RoleEnum: string
{
    case Lider          = "lider";
    case Admin          = "administrador";
    case Agremiado      = "agremiado";
    case NoAgremiado    = "no_agremiado";
    case Finanzas       = "finanzas";
}
```

El rol por defecto al crear un usuario es `no_agremiado`:

```sql
role VARCHAR(20) NOT NULL DEFAULT 'no_agremiado'
```

## Aplicación del control de acceso

El control de acceso se aplica en dos capas:

### 1. Middleware de autenticación

`AuthMiddleware` verifica que exista una sesión activa. Si no, lanza `UnauthorizedException` y el `MiddlewareRunner` redirige a `/cuentas/login.php`:

```php
$runner->runOrRedirect($middleware->auth());
```

### 2. Middleware de roles

`RoleMiddleware` verifica que el rol del usuario en sesión sea uno de los roles permitidos. Si está autenticado pero no tiene el rol, lanza `ForbiddenException` y redirige a `/portal/acceso-denegado.php` (HTTP 403):

```php
$runner->runOrRedirect(
    $middleware->role(RoleEnum::Finanzas, RoleEnum::Admin)
);
```

El `MiddlewareFactory` encapsula la creación de ambos middlewares:

```php
final readonly class MiddlewareFactory
{
    public function auth(): AuthMiddleware { ... }
    public function role(RoleEnum ...$roles): RoleMiddleware { ... }
}
```

<Warning>
  Si un usuario autenticado intenta acceder a un recurso sin el rol necesario, recibe HTTP 403 y es redirigido a la página de acceso denegado. No se revela información sobre el recurso.
</Warning>

## Tabla de roles en la base de datos

```sql
CREATE TABLE IF NOT EXISTS users (
    user_id  INT AUTO_INCREMENT PRIMARY KEY,
    email    VARCHAR(100) NOT NULL UNIQUE,
    role     VARCHAR(20)  NOT NULL DEFAULT 'no_agremiado',
    active   BOOLEAN               DEFAULT TRUE,
    -- ...
);
```

El campo `active` permite desactivar un usuario sin eliminarlo. Un usuario inactivo no puede iniciar sesión aunque sus credenciales sean válidas.
