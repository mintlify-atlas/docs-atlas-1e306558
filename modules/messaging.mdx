## Descripción general

`MessagingModule` extiende `AbstractModule` sin registraciones DI adicionales. La infraestructura de mensajería se apoya en la tabla `mail_queue` para gestionar el envío de correos de forma asíncrona, y en los enumerados de dominio que clasifican los mensajes del buzón interno.

```php
final class MessagingModule extends AbstractModule {}
```

---

## Buzón interno

El buzón (`/portal/buzon/`) permite a los agremiados enviar mensajes a la administración del sindicato. Los mensajes se clasifican por tipo, prioridad y estado.

### Tipos de mensaje (`MessageTypeEnum`)

```php
enum MessageTypeEnum: string
{
    case GeneralContact      = "contacto_general";
    case TransparencyDoubt   = "transparencia_duda";
    case ComplaintBox        = "buzon_queja";
}
```

| Caso | Valor | Descripción |
|------|-------|-------------|
| `GeneralContact` | `contacto_general` | Consultas o comunicados generales al sindicato |
| `TransparencyDoubt` | `transparencia_duda` | Dudas relacionadas con la sección de transparencia |
| `ComplaintBox` | `buzon_queja` | Quejas formales ante la directiva |

### Prioridades (`MessagePriorityEnum`)

```php
enum MessagePriorityEnum: string
{
    case Low    = "baja";
    case Medium = "media";
    case High   = "alta";
    case Urgent = "urgente";
}
```

### Estados de conversación (`MessageStateEnum`)

```php
enum MessageStateEnum: string
{
    case Open       = "abierto";
    case InProgress = "en_proceso";
    case Closed     = "cerrado";
    case Answered   = "respondido";
    case Archived   = "archivado";
}
```

## Sistema de correo electrónico

### Cola de envío (`mail_queue`)

Todos los correos salientes del sistema (recuperación de contraseña, notificaciones de préstamo, respuestas del buzón) se encolan en `mail_queue` antes de ser enviados. Un proceso cron procesa la cola de forma periódica.

```sql
CREATE TABLE IF NOT EXISTS mail_queue (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    recipient    TEXT         NOT NULL,
    subject      VARCHAR(255) NOT NULL,
    body         MEDIUMTEXT   NOT NULL,    -- HTML del correo
    alt_body     TEXT,                     -- texto plano
    priority     TINYINT      DEFAULT 2,  -- 1: Urgente, 2: Normal, 3: Newsletter
    attempts     INT          DEFAULT 0,
    max_attempts INT          DEFAULT 3,
    status       ENUM('pending','sending','sent','failed') DEFAULT 'pending',
    last_error   TEXT,
    scheduled_at TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    locked_at    TIMESTAMP    NULL,
    lock_token   VARCHAR(64)  NULL,        -- evita colisiones entre procesos
    created_at   TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    INDEX (status, priority, scheduled_at),
    INDEX (status, lock_token)
);
```

**Estados de un correo en cola:**

| Estado | Descripción |
|--------|-------------|
| `pending` | En espera de ser procesado |
| `sending` | Siendo enviado por el proceso cron (bloqueado con `lock_token`) |
| `sent` | Enviado exitosamente |
| `failed` | Falló tras agotar `max_attempts` (por defecto 3) |

### PasswordRecoveryMailer

Implementación concreta de `PasswordRecoveryNotifierInterface` que usa `MailerInterface` (PHPMailer) y renderiza plantillas Latte:

```php
final readonly class PasswordRecoveryMailer implements PasswordRecoveryNotifierInterface
{
    public function __construct(
        private MailerInterface   $mailer,
        private RendererInterface $renderer,
        private string            $templateBasePath,
    ) {}

    public function sendMagicLink(string $email, string $magicLink): void
    {
        $subject      = "Recuperación de contraseña";
        $templatePath = $this->templateBasePath
            . "/templates/emails/reset-password.latte";

        $body    = $this->renderer->renderToString($templatePath, [
            "link" => $magicLink,
            "year" => date('Y'),
        ]);
        $altBody = "Abre este enlace para recuperar tu contraseña: {$magicLink}";

        $this->mailer->send([$email], $subject, $body, $altBody);
    }
}
```

### Prioridades de la cola

| Valor | Significado |
|-------|-------------|
| `1` | Urgente — se procesa primero |
| `2` | Normal — comportamiento por defecto |
| `3` | Newsletter / masivo — menor prioridad |

<Warning>
El campo `lock_token` y `locked_at` en `mail_queue` implementan un mecanismo de bloqueo optimista para evitar que múltiples procesos cron envíen el mismo correo simultáneamente. Un correo bloqueado por más de 5 minutos debe considerarse huérfano y puede liberarse manualmente.
</Warning>
